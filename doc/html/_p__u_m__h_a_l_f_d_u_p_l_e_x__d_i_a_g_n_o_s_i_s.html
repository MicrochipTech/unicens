HalfDuplex Diagnosis

Introduction

HalfDuplex Diagnosis is performed on the TimingMaster node. It inspects the ring segment by segment until it finds a ring break or detects a closed ring.

It cannot be performed during Node Discovery or Programming service.

Operation

Calling Ucs_Diag_StartHdxDiagnosis() starts the HalfDuplex Diagnosis. The function has a parameter report_fptr. It points to the report callback function of type Ucs_Diag_HdxReportCb_t. report_fptr will deliver a result for each inspected node as well as a result which indicates the end of the diagnosis session.

Parameter code Parameter cable_diag_result Parameter position Signature Description
UCS_HDX_RES_SUCCESS (1) segment number x Tested segment is ok.
UCS_HDX_RES_SLAVE_WRONG_POS (1) segment number x Answer came from wrong position.
UCS_HDX_RES_RING_BREAK (1) segment number x Ring break detected.
UCS_HDX_RES_NO_RING_BREAK (1) segment number x Ring is closed.
UCS_HDX_RES_NO_RESULT (1) segment number x Diagnosis reported an unexpected error.
UCS_HDX_RES_TIMEOUT 0xFF UCS_HDX_DUMMY_POS dummy An expected result message did not arrive in time.
UCS_HDX_RES_ERROR (1), 0xFF UCS_HDX_DUMMY_POS dummy An unexpected error occurred.
UCS_HDX_RES_END 0xFF UCS_HDX_DUMMY_POS dummy HalfDuplex Diagnosis ended.

(1) value as delivered from INIC. For interpretation see INIC manual.

So the chronological order of results will be:

  • 0 ... N-1 times UCS_HDX_RES_SUCCESS. N is the number of nodes in the ring.
  • one of the results
    • UCS_HDX_RES_SLAVE_WRONG_POS
    • UCS_HDX_RES_RING_BREAK
    • UCS_HDX_RES_NO_RING_BREAK
    • UCS_HDX_RES_NO_RESULT
    • UCS_HDX_RES_TIMEOUT
    • UCS_HDX_RES_ERROR
  • and UCS_HDX_RES_END to indicate the end of the HalfDuplexDiagnosis

Parameter position denotes the currently tested segment. Numbering starts with 1 which is the segment between node 0 (= root node) and node 1.

Signature information comes from the first node of the tested segment. When testing segment n (between node n - 1 and node n) the reported signature comes from node n - 1. Signature value dummy means that the signature parameter points to NULL. In this case no valid signature is available.

Please note: The HalfDuplex Diagnosis cannot be started as long as Node Discovery, FullDuplex Diagnosis or Programming is performed.
