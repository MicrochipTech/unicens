FullDuplex Diagnosis

Description

The FullDuplex Diagnosis performs the following actions:

  • Check device presence.
  • Gather diagnosis relevant information from all devices in the network, including node address, MAC address, device identifier, number of ports and status of the physical connections.
  • Trigger a cable link diagnosis (if required) to verify the physical linkage.

Definition, Limits

FullDuplex Diagnosis supports networks where the TimingMaster can be a single port or a multi port INIC. The TimingSlaves can be single port or dual port INICs. The INICs belonging to one certain port of the TimingMaster are called a branch. Two INICs which are directly connected are called a segment. Segment numbers start with 1, i.e. the connection between the Timingmaster and the first INIC of a segment has number 1.

Usage

API

The application starts the FullDuplex Diagnosis by calling Ucs_Diag_StartFdxDiagnosis(). Its parameter report_fptr is a callback function of type Ucs_Diag_FdxReportCb_t.

This callback function is called on every explored segment, on an unexpected error and when the FullDuplex Diagnosis finishes. The callback function's parameter result is of type Ucs_Fdx_Report_t and reports the result of the segment which was explored at last.

The application can stop a running FullDuplex Diagnosis by calling Ucs_Diag_StopFdxDiagnosis().

Please note: The FullDuplex Diagnosis cannot be started as long as Node Discovery, HalfDuplex Diagnosis or Programming is performed.

Result

result.code determines the type of the result:

UCS_FDX_TARGET_FOUND

A working segment was found.
result.segment contains a detailed description of the segment.

UCS_FDX_CABLE_LINK_RES

A Cable Link Diagnosis was executed. Exploration on the current branch finishes. If there are remaining branches FullDuplex Diagnosis will continue and explores the next branch.
The result of the Cable Link Diagnosis is reported via result.cable_link_info.

 This attribute corresponds with the parameter Result of the INIC function ExtendedNetworkControl.CableLinkDiagnosis.

UCS_FDX_ERROR

An unexpected behaviour occurred. result.err_info specifies the error. Valid values are defined in Ucs_Fdx_ErrCode_t.
Hints:

  • UCS_FDX_ERR_PORT_NOT_USED and UCS_FDX_ERR_NO_FDX_MODE will stop working on the current branch only. The FullDuplex Diagnosis will try to explore other branches. All other error results will lead to a stop of the FullDuplex Diagnosis.
  • UCS_FDX_ERR_STOP_DIAG_FAILED or UCS_FDX_ERR_TERMINATED the INIC may remain in FullDuplex Diagnosis mode after the EHC has finished the FullDuplex Diagnosis. A reset of the INIC may resolve this situation.

UCS_FDX_ABORTED

The Application stopped the FullDuplex Diagnosis by calling Ucs_Diag_StopFullDuplexDiagnosis().

UCS_FDX_FINISHED
The FullDuplex Diagnosis terminated. The FullDuplex Diagnosis ends always with this result.
